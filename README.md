# Retail Data Analytics & Engineering Pipeline

## Project Overview
This project demonstrates an end-to-end data engineering and analytics pipeline, transforming raw retail transaction data into actionable business intelligence. By bridging data engineering principles with business strategy, this project uncovers critical customer purchasing patterns, demographic trends, and product performance metrics.

## The Tech Stack
* **Python (Pandas):** Data cleaning, preprocessing, and handling inconsistencies.
* **PostgreSQL:** Relational database management, complex querying, and KPI extraction.
* **Power BI:** Interactive data visualization and dynamic business dashboards.

## Key Achievements & Business Impact
* **Systematized** a highly efficient automated data pipeline using **Python (Pandas)** to clean and preprocess **3.9K+** retail transactions, ensuring robust data integrity for a seamless downstream **PostgreSQL** database migration.
* **Engineered** a scalable, relational **PostgreSQL database** to efficiently execute complex queries and analytical aggregations, successfully extracting **critical business KPIs** regarding gender-based revenue splits and overall product performance trends.
* **Synthesized** actionable **cross-functional insights** through an interactive **Power BI** dashboard, directly identifying a **73%** non-subscriber growth opportunity and calculating a **$59.76** average transaction value for key business stakeholders.

## Repository Contents
* `data_cleaning.ipynb`: The Python script used to clean and format the raw CSV data.
* `queries.sql`: The PostgreSQL queries used to answer specific business questions and aggregate data.
* `customer_behavior_dashboard.pbix`: The interactive Power BI dashboard file.
* `dataset.csv`: The cleaned data ready for SQL ingestion.
